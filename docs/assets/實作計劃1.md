這是一份為您整理的**完整且經過 HPC 最佳化**的 DG (Discontinuous Galerkin) 單一元素平流方程式實作藍圖。這份藍圖整合了無分支 (Branchless) 邊界通量以及具備精準收尾機制的 LSRK54 時間迭代。

您可以直接對照這個架構來撰寫與組織您的程式碼：

### 第一階段：初始化與前置準備 (Initialization)
在進入時間迴圈之前，您需要準備好所有不隨時間變動的靜態變數與矩陣：
1. **幾何度量與網格資訊**：
   - 定義三角形頂點座標。計算 Jacobian 行列式 $|J|$ 與幾何度量係數（$rx, ry, sx, sy$），用於將參考單元導數轉換為物理空間導數。
   - **計算邊界外向單位法向量**：針對三角形的三條邊，計算其對應的單位外向法向量 $\boldsymbol{n} = (n_x, n_y)$。
2. **算子載入**：
   - 載入參考單元的微分矩陣 $D_\xi, D_\eta$。
   - 載入邊界提取矩陣 $E$ 與權重矩陣 $W, W^e$。
3. **設定物理場與初始條件**：
   - 設定平流速度 $\boldsymbol{V} = (u, v)$，且滿足 $u=v$ 以確保場的靜態特性。
   - 初始化數值解 $q^0 = x - y$。**明確聲明：在此實驗中，精確解 qexact​=x−y 恆不隨時間改變。**
4. **計算時間步長 Δt**：
   - $\Delta t = \text{CFL} \cdot \frac{h_{\min}}{|\boldsymbol{V}|_{\max} \cdot N^2}$。

---

### 第二階段：核心 RHS 計算函數 (The `compute_rhs` Function)

這是一個獨立的函數，輸入當前的數值解 $q$，輸出時間變化率 $dq/dt$。

**函數輸入**：`q` (當前時間步的內部數值解向量)
**函數輸出**：`rhs` (時間變化率向量)

**實作步驟**：
1. **座標轉換 (Chain Rule)**：
   - 利用幾何度量計算物理空間微分矩陣：$D_x = rx \cdot D_\xi + sx \cdot D_\eta$ 以及 $D_y = ry \cdot D_\xi + sy \cdot D_\eta$。
1.  **計算體積項 (Volume Terms)**：
    * 計算 $D_x (u \odot q) + D_y (v \odot q)$。
    * 計算 $u \odot (D_x q) + v \odot (D_y q)$。
    * 計算 $(D_x u + D_y v) \odot q$。
    * 將上述三項相加並乘上 $-\frac{1}{2}$，得到 `volume_term`。
2.  **計算邊界懲罰項 (Surface / Penalty Term)**：
    * 使用提取矩陣 $E$ 抓取邊界上的數值解 $q_{bound}$。
    * 計算邊界上的精確解 $q_g = x_{bound} - y_{bound}$ (因為此案例外部界條件恆定)。
    * 計算邊界波速投影：$s = \boldsymbol{n} \cdot \boldsymbol{V}_{bound}$。
    * **無分支迎風通量 (Branchless Upwind Flux)**：計算懲罰向量 $p$：
        $$p_i = \frac{1}{2}(s_i - |s_i|)(q_{bound,i} - q_{g,i})$$
    * 組裝表面項矩陣乘法：`surface_term = (1 / |T|) * W_inv @ E_T @ W_e @ p`。
3.  **回傳總和**：
    * `return volume_term + surface_term`

### 第三階段：LSRK54 時間迭代 (Time Integration)

使用低記憶體消耗的 Runge-Kutta 演算法推進時間，並確保精確停在 $t_{final}$。

```python
# 預先定義 RK 係數
A_RK = np.array([0.0,
                 -567301805773.0/1357537059087.0,
                 -2404267990393.0/2016746695238.0,
                 -3550918686646.0/2091501179385.0,
                 -1275806237668.0/842570457699.0])
B_RK = np.array([1432997174477.0/9575080441755.0,
                 5161836677717.0/13612068292357.0,
                 1720146321549.0/2090206949498.0,
                 3134564353537.0/4481467310338.0,
                 2277821191437.0/14882151754819.0])

# 初始設定
t = 0.0
t_final = 1.0  # 依您的需求設定
tol = 1e-12

# q 已經在第一階段初始化為 q_initial
du = np.zeros_like(q)

while t < t_final - tol:
    # 動態步長檢查：防止最後一步超出 t_final
    current_dt = min(dt, t_final - t)

    # 每個 Time step 開始前，清空殘差暫存
    du.fill(0.0)

    # LSRK54 的 5 個 Stage
    for i in range(5):
        # 1. 取得 RHS
        R_q = compute_rhs(q)

        # 2. 更新殘差 du
        du = A_RK[i] * du + current_dt * R_q

        # 3. 更新數值解 q
        q = q + B_RK[i] * du

    # 推進時間
    t += current_dt
```

### 第四階段：誤差驗證 (Error Evaluation)

當時間迴圈結束後，計算離散 $L_2$ 誤差來驗證演算法正確性。

1.  **取得最終精確解**：計算在所有內部節點上的 $q_{exact} = x - y$。
2.  **計算誤差向量**：$e = q - q_{exact}$。
3.  **計算加權 $L_2$ 誤差**：
    * 質量矩陣 (Mass Matrix) 近似為 $M = |T| \cdot W$ (對角矩陣)。
    * 計算誤差：$$\text{Error}_{L_2} = \sqrt{ |T| \cdot \sum_{i} (e_i^2 \cdot W_{i,i}) }$$
    *(註：若誤差落在 $10^{-10}$ 到機器精度 $10^{-15}$ 之間，代表您的 DG 空間與時間離散皆已完美實作！)*
